Preguntas de lectura: Capítulo 1

1. Responda brevemente, ¿Qué son los datos? ¿Qué es una red de datos? ¿Qué
es una red convergente?

Datos: los datos son información  y que se utiliza para una variedad de propósitos en nuestra vida cotidiana.Los datos son fragmentos de información que se transmite a través de las redes de datos. Estos datos pueden ser números, palabras u otros tipos de información, Los datos individualmente no tienen significado, estos deben estar organizados y tener un contexto para que pueda ser de utilidad para las personas

Red de datos: Una red de datos es una agrupación de dispositivos que están conectados entre sí con el objetivo de facilitar la comunicación y el intercambio de información. Estos dispositivos pueden ser computadoras, servidores, enrutadores, conmutadores, dispositivos móviles y otros dispositivos de red. Las redes de datos pueden variar en tamaño y capacidad, pero todas comparten cuatro elementos fundamentales:

	-Reglas y acuerdos que regulan cómo se envían, redireccionan, reciben e interpretan los mensajes.
	-Los mensajes o unidades de información que viajan de un dispositivo a otro.
	-Un medio de interconexión que permite el transporte de los mensajes entre los dispositivos.
	-Los propios dispositivos de la red que se encargan de intercambiar los mensajes entre sí.

red convergente: En una red convergente, todos los servicios, como voz, video y datos, pueden transmitirse a través de una única plataforma. En el pasado, cada servicio requería tecnologías diferentes y reglas específicas para su transmisión. Sin embargo, gracias a los avances tecnológicos, ahora es posible consolidar todas estas redes dispersas en una sola plataforma: una red convergente. Aunque aún existen puntos de contacto y dispositivos especializados, se utiliza una infraestructura de red común.

2. Describir las características de las arquitecturas de red: tolerante a fallas, escalabilidad, calidad del servicio y seguridad. Mostrar sus respectivos ejemplos.

2.1. Tolerante a fallas: una red tolerante a fallas es una red que limita el impacto de las fallas de software o hardware y permite una rápida recuperación de dichas fallas. Estas redes se basan en enlaces o rutas redundantes entre las fuentes y los destinos de los mensajes. Si falla un enlace o una ruta, estos procesos garantizan que los mensajes se redirijan inmediatamente a otro enlace que sea transparente para los usuarios en ambos extremos. Tanto la infraestructura física como los procesos lógicos para enrutar mensajes en la red están diseñados para adaptarse a esta redundancia.Un ejemplo de una red tolerante a fallas es una red de telefonía celular que usa múltiples torres de telefonía para garantizar que los usuarios puedan realizar llamadas y enviar mensajes de texto, incluso si una torre de telefonía celular no funciona.

2.2. Escalabilidad: la escalabilidad se refiere a la capacidad de una red para crecer y adaptarse a medida que aumenta la cantidad de usuarios y dispositivos conectados. Una red escalable debe poder manejar más tráfico sin degradar el rendimiento o la calidad del servicio. Un ejemplo de una red escalable es Internet, que ha crecido rápidamente desde sus inicios y todavía es capaz de atender a un número cada vez mayor de usuarios y dispositivos conectados.

2.3. Calidad de servicio: la calidad de servicio se refiere a la capacidad de una red para garantizar que los mensajes se entreguen a tiempo y cumplan con los requisitos de rendimiento especificados. La calidad del servicio es especialmente importante para aplicaciones que requieren transferencia de datos en tiempo real, como videoconferencias o juegos en línea. Un ejemplo de una red que prioriza la calidad del servicio es una red de telefonía de Voz sobre IP (VoIP) que utiliza tecnología de priorización de tráfico para garantizar que las llamadas de voz se entreguen de manera confiable y puntual, incluso en condiciones de gran congestión de la red.

2.4 Seguridad: La seguridad se refiere a la capacidad de la red para proteger los datos y la información del usuario contra accesos no autorizados o ataques malintencionados. Las redes deben diseñarse para proteger la privacidad y la integridad de los datos y garantizar que solo los usuarios autorizados puedan acceder a la información. Un ejemplo de una red segura es una red privada virtual (VPN), que utiliza cifrado para proteger los datos transmitidos a través de la red y garantizar que solo los usuarios autorizados puedan acceder a esa información.

3. Investigar qué son las redes orientadas y no orientadas a la conexión.

Las redes orientadas a la conexión y las redes no orientadas a la conexión son dos tipos de arquitecturas de red que se utilizan para transmitir datos a través de una red de computadoras.Las redes orientadas a la conexión, también conocidas como redes de circuitos virtuales, establecen una conexión dedicada entre dos dispositivos antes de transferir datos. En este tipo de red, las rutas de comunicación entre dispositivos se establecen antes de que se transfieran los datos. Una vez que se establece la conexión, la información viaja a lo largo de este camino especial. Las redes orientadas a la conexión son útiles para aplicaciones que requieren una transferencia de datos fiable y en tiempo real, como la telefonía y las videoconferencias.
Por otro lado, las redes sin conexión, también conocidas como redes de datagramas, no establecen una conexión específica antes de transferir datos. En este tipo de red, los datos se dividen en paquetes y se envían de forma independiente a través de la red. Cada paquete se enruta de forma independiente a través de la red y puede tomar una ruta diferente hasta su destino. Las redes sin conexión son útiles para aplicaciones que no requieren transferencia de datos en tiempo real, como el correo electrónico y la navegación web. 

4. Describa que es la calidad del servicio y responda: ¿Por qué importa la calidad del servicio en una red de datos?

La calidad de servicio (QoS) se refiere a la capacidad de una red para entregar datos a tiempo y cumplir con ciertos requisitos de rendimiento. QoS puede incluir priorizar ciertos tipos de tráfico, limitar la congestión de la red y proporcionar un ancho de banda mínimo para ciertas aplicaciones. 

La calidad del servicio es fundamental en las redes de datos porque permite que las aplicaciones críticas, como las videoconferencias y la telefonía de voz sobre IP (VoIP), funcionen de manera confiable y sin interrupciones. Si la red no puede garantizar QoS, estas aplicaciones pueden experimentar latencia, pérdida de paquetes o mala calidad de audio y video, lo que afecta negativamente la productividad y la satisfacción del usuario. Además, QoS también es importante para una transmisión de datos fiable y segura. Si, por ejemplo, se transmite información confidencial, como datos de tarjetas de crédito o información médica, es importante que la red garantice que estos datos estén seguros y se transmitan sin interrupciones.


Preguntas de lectura: Capítulo 2

1. Describir la estructura de una red, incluidos los dispositivos, medios y servicios necesarios para lograr comunicaciones exitosas:

La estructura de la red incluye los dispositivos, medios y servicios necesarios para una comunicación exitosa. Los dispositivos son elementos físicos o hardware en una red, como computadoras, conmutadores, enrutadores, servidores, impresoras, etc. Los portadores de datos son cables, fibras ópticas, ondas de radio y otros medios utilizados para conectar dispositivos y permitir la transferencia de datos. Los servicios son programas de comunicación denominados software que se ejecutan en dispositivos conectados a una red. Estos servicios incluyen muchas aplicaciones web comunes que las personas usan todos los días, como alojamiento de correo electrónico y alojamiento web. Además, existen reglas y procesos que rigen la comunicación de red, así como herramientas y comandos para construir y mantener redes. La combinación de estos componentes permite que los mensajes se envíen de forma segura al destino correcto.

2. Comparar y contrastar los siguientes términos: RED, LAN, WAN,
INTERNETWORK E INTERNET.

- Red: Este es un término general que se refiere a cualquier conjunto de dispositivos y medios utilizados para comunicarse entre sí. Puede ser LAN o WAN.

- LAN: una red de área local utilizada para conectar dispositivos dentro de un área geográfica limitada, como una oficina, un edificio o un campus. Las LAN suelen ser propiedad de organizaciones y se utilizan para compartir recursos como archivos, impresoras y aplicaciones.

- WAN: Es una red de área amplia que se utiliza para conectar dispositivos en un área geográfica grande, como una ciudad, un país o un continente. Las WAN se utilizan para conectar diferentes LAN y permitir la comunicación entre ellas. 

- Internetwork: Es una red formada por varias redes interconectadas. Por ejemplo, una empresa puede tener varias LAN conectadas a una WAN para formar una red de Internet.

- Internet: Es una red global de computadoras interconectadas que se comunican entre sí usando el Protocolo de Internet (IP). Internet es una red pública para compartir información y recursos en todo el mundo. 

En pocas palabras, una red es un término general para cualquier conjunto de dispositivos y medios utilizados para comunicarse entre sí. Una LAN es una red de área local que se usa para conectar dispositivos dentro de un área geográfica limitada, mientras que una WAN es una red de área amplia que se usa para conectar dispositivos dentro de un área geográfica más grande. INTERNETWORK es una red de redes interconectadas, mientras que INTERNET es una red global de computadoras interconectadas que se comunican entre sí utilizando el Protocolo de Internet (IP).

3. ¿Cuál es la diferencia entre INTERNETWORK E INTERNET?

La principal diferencia entre INTERNETWORK e INTERNET es que INTERNETWORK es una red que se compone de varias redes interconectadas, mientras que INTERNET es una red global de computadoras interconectadas que utilizan el protocolo de Internet (IP) para comunicarse entre sí.

Un INTERNETWORK es una red que se compone de varias redes interconectadas, lo que significa que es una red que conecta varias redes locales (LAN) y redes de área amplia (WAN) para formar una red más grande. Por ejemplo, una empresa puede tener varias LAN interconectadas mediante una WAN para formar un internetwork.

Por otro lado, el INTERNET es una red global de computadoras interconectadas que utilizan el protocolo de Internet (IP) para comunicarse entre sí. El INTERNET es una red pública que se utiliza para compartir información y recursos en todo el mundo. A diferencia de un INTERNETWORK, el INTERNET no es propiedad de ninguna organización y está disponible para su uso público.

4. Describir la diferencia entre Tarjeta de interfaz de red (NIC), puerto físico e
interfaz de red.

- Tarjeta de interfaz de red (NIC): un dispositivo de hardware utilizado para conectar una computadora u otro dispositivo host a una red. Una NIC proporciona una conexión física a la red y se conecta directamente a una PC u otro dispositivo host. 

- Puerto físico: un enchufe o conector en un dispositivo de red a través del cual los medios se conectan a una computadora host u otro dispositivo de red. Los puertos físicos se utilizan para conectar dispositivos a la red y permitir la transferencia de datos. 

- Interfaz de red: este es un puerto dedicado en un dispositivo conectado a Internet conectado a cada red. Los enrutadores se utilizan para conectar redes y los puertos de los enrutadores se denominan interfaces de red. Las interfaces de red se utilizan para conectar diferentes redes y permitir la comunicación entre ellas.

la NIC es un dispositivo de hardware que se utiliza para conectar una computadora u otro dispositivo host a una red, mientras que el puerto físico es un conector o toma en un dispositivo de red en el cual el medio se conecta con un host o con otro dispositivo de red. La interfaz de red es un puerto especializado de un dispositivo de internetworking que se utiliza para conectar diferentes redes y permitir la comunicación entre ellas.

5. ¿Por qué se dice que un protocolo es independiente de la tecnología?

Se dice que un protocolo es independiente de la tecnología porque describe solamente qué funciones se requieren de una regla de comunicación en particular, pero no cómo realizarlas. En otras palabras, un protocolo describe lo que se debe hacer, pero no cómo se debe hacer. 

Por ejemplo, el protocolo HTTP (Hypertext Transfer Protocol) se utiliza para la comunicación entre servidores web y navegadores web. HTTP no especifica qué lenguaje de programación se utiliza para crear el explorador, qué software de servidor web se debe utilizar para servir las páginas web, sobre qué sistema operativo se ejecuta el software o los requisitos necesarios para mostrar el explorador. Tampoco describe cómo detecta errores el servidor, aunque sí describe qué hace el servidor si se produce un error.

ya que los protocolos describen solamente qué funciones se requieren de una regla de comunicación en particular pero no cómo realizarlas, es posible que la implementación de un protocolo en particular sea independiente de la tecnología. Esto significa que una computadora y otros dispositivos, como teléfonos móviles o PDA, pueden acceder a una página web almacenada en cualquier tipo de servidor web que utilice cualquier tipo de sistema operativo desde cualquier lugar de Internet.

6. Consultar y explicar brevemente qué es una Unidad de Datos del Protocolo (PDU).

Una PDU es una sección de datos que se encapsula en cada capa del modelo OSI o TCP/IP y que tiene un nombre distinto en cada capa. La PDU es la unidad básica de datos que se transmite a través de una red y que se utiliza para proporcionar servicios de comunicación entre dispositivos de red. En el modelo OSI, cada capa agrega su propia información de encabezado a la PDU que recibe de la capa superior. En el modelo TCP/IP, la PDU se denomina de manera diferente en cada capa. Por ejemplo, en la capa de aplicación, la PDU se llama "datos", mientras que en la capa de transporte, la PDU se llama "segmento". En la capa de internetwork, la PDU se llama "paquete", y en la capa de acceso a la red, la PDU se llama "trama".
